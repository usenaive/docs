The `tools` command lets you explore all 54 available tools without leaving the terminal. It reads from the local YAML tool definitions bundled with the SDK — no API key or network access required.

## List All Tools

```bash
naive tools
```

Displays every tool grouped by category:

## Tool Help {#help}

```bash
naive tools help <name>
```

Get the full documentation for a specific tool, including description, parameters, output schema, hints, and error recovery.

### Example

```bash
naive tools help send_email
```

```text
  Send Email (send_email)
  Category: communication (core)
  Tags: email, send, outbound, resend, communication

  Send an email from the agent's provisioned email address via Resend.

  Sends an outbound email from the agent's assigned email address.
  The email is delivered via the Resend transactional email service.

  The agent must have a provisioned email address (set up via domain
  verification). If no address is configured, the call fails with
  `no_email_address`.

  A `reason` field is required — it is logged for audit purposes and
  helps the user understand why the email was sent.

  Example:
    send_email(
      to: "alice@example.com",
      subject: "Weekly report",
      body: "Hi Alice, here is your weekly summary…",
      reason: "Sending scheduled weekly report to stakeholder"
    )

  Input (* = required):
    * to                   string     Recipient email address.
    * subject              string     Email subject line.
    * body                 string     Plain-text email body.
    * reason               string     Audit reason explaining why the email is being sent.

  Output:
      sent                 boolean    Whether the email was accepted for delivery.
      emailId              string     Resend message ID, or null if sending failed.

  Hints (after success):
    → list_inbox: Check for replies in the inbox.

  On success: Email queued for delivery. Use list_inbox to monitor for replies.
  Error recovery:
    no_email_address: The agent does not have a provisioned email address.
    rate_limited: Too many emails sent in a short window.
    blocked_sensitive: The email body was flagged for containing sensitive content.
```

<Note>
  Tool help is loaded from YAML definitions bundled with the SDK. It works completely offline — no API key or network connection needed. This is the same documentation returned by `naive_tool_help` in the MCP server.
</Note>
