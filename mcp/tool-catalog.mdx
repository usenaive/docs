The Naive platform provides **98 tools** across two layers: **54 primitives** (employee-level tools for doing work) and **44 platform orchestration tools** (user-level tools for managing the platform, coming soon). Of the primitives, 8 are registered as [core tools](/mcp/core-tools) in the MCP context window; the remaining 46 are accessible via the [meta-tools](/mcp/meta-tools) discovery flow.

<Note>
  **Core** tools (marked ★) are callable directly. **Extended** tools require `naive_search → naive_tool_help → naive_execute`. See the [API reference](/api-reference) for full parameter documentation.
</Note>

## Platform Orchestration <sup>Coming Soon</sup>

<Info>
  The Platform Orchestration tools are **coming soon**. They will be available in the MCP server when running in **orchestration mode** and use **user-level keys** (`nv_pk_...`) instead of employee keys.
</Info>

These tools manage the Naive platform itself — companies, employees, tasks, deliverables, loops — rather than acting as an employee. They are a separate layer from the primitives above.
