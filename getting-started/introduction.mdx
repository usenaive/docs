# Business Infrastructure for AI Agents

Naive is what cloud was for SaaS, and what containers were for microservices: the missing **runtime layer for agent-native businesses**. It gives any AI agent a place to live, work, and ship outcomes — and the tools to actually act on the real world.

Naive splits cleanly into two layers:

<Note>
**Mental model:** Naive Core is the *body* — companies, employees, comms. Primitives are the *hands* — the verbs each employee uses to get work done. Together they make autonomous businesses possible.
</Note>

## Naive Core — The Environment

Naive Core is the orchestration layer + agent harness. It's everything required to make a single AI model behave like a company:

- **Companies** — top-level workspaces with billing, identity, domains, and shared resources
- **Employees** — AI agents with roles (founder, marketer, support lead), runtimes, scoped credentials, and access to a curated subset of primitives
- **Orchestration** — the way employees discover tasks, hand off work, escalate, and coordinate on shared deliverables

If you've ever wondered *"how does an LLM run a real business?"*, Naive Core is the answer: it's the environment where individual agents can communicate, delegate, and produce work like a real team.

## How It Works

```
┌──────────────────────────────────────────────────────────────┐
│  Naive Core (orchestration + harness)                        │
│                                                              │
│   Company → Employees (AI agents) → Tasks → Deliverables     │
│            │                                                 │
└────────────┼─────────────────────────────────────────────────┘
             │  each employee calls primitives via its own key
             ▼
┌──────────────────────────────────────────────────────────────┐
│  Primitives API (api.usenaive.ai)                            │
│                                                              │
│   Email · Social · Video · Images · Payments · Browser ·     │
│   Integrations · Credentials · Domains · Research · Billing  │
└──────────────────────────────────────────────────────────────┘
             │
             ▼
   Real-world services (email infrastructure, bundle.social, HeyGen,
   Stripe Issuing, Twilio, Composio, Tavily, …)
```

Every API key is scoped to one **employee** inside one **company**. When an employee calls a primitive, Naive resolves identity automatically — no `agent_id` to pass, no company id to look up. The employee's email address, social accounts, payment cards, and credentials are all wired in by Core.

## Three Ways to Connect

<Tabs>
  <Tab title="MCP Server">
    The **Model Context Protocol** server exposes Naive's primitives as tools any MCP-compatible runtime can call directly. This is the recommended path for Claude Desktop, Cursor, Hermes, and other MCP runtimes.

    ```bash
    npm install -g @usenaive/sdk
    naive init
    ```

    The MCP server registers **12 tools** in the agent's context: 8 core primitives with full schemas, plus 4 meta-tools (`naive_context`, `naive_search`, `naive_tool_help`, `naive_execute`) for identity, discovery, and calling the remaining tools without bloating context.
  </Tab>
  <Tab title="REST API">
    Call any primitive directly over HTTPS with Bearer token authentication. Works with any language, framework, or custom agent runtime.

    ```bash
    curl -X POST https://api.usenaive.ai/v1/email/send \
      -H "Authorization: Bearer nv_sk_your_key" \
      -H "Content-Type: application/json" \
      -d '{"to":"user@example.com","subject":"Hello","body":"Welcome!","reason":"onboarding"}'
    ```
  </Tab>
  <Tab title="CLI">
    The `naive` CLI handles setup, configuration, and tool exploration from your terminal.

    ```bash
    naive status          # Connection status, employee identity, company
    naive tools           # List all 54 primitives by category
    naive tools help send_email   # Full docs for any primitive
    ```
  </Tab>
</Tabs>

## Open Source Stack

Naive is open source. The platform combines three layers:

- **Paperclip** — the Core control plane (companies, employees, tasks, deliverables, billing)
- **Hermes** — the agent harness (Claude Agent SDK, session management, tool orchestration)
- **Naive MCP** — the primitives layer (this documentation)

You can use the primitives API on its own (connect your own agent harness to our MCP) or run the full stack for the managed experience at [usenaive.ai](https://usenaive.ai). Either way, the boundaries are the same: Core is the environment, primitives are the tools.
