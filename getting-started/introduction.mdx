# Primitives Infrastructure for AI Agents

Naive provides the real-world primitives that AI agents need to actually run businesses and complete work — sending emails, posting to social media, creating videos, making purchases, browsing the web, and connecting to third-party services.

We handle the primitives layer. You bring the agent runtime.

<Note>
**Naive is being open-sourced.** Our platform provides a managed experience where users select and run hosted agent runtimes (similar to Cursor) through a chat interface. The runtime connects to Naive's primitives MCP to let agents create and run companies autonomously. The primitives API documented here is the foundation that powers all of it.
</Note>

## What Are Primitives?

Primitives are the real-world capabilities that agents can't do on their own — they need infrastructure to send an actual email, post to an actual Twitter account, charge an actual credit card. Naive provides 54 of these primitives across 11 categories, accessible through a single API key.

## How It Works

```
Your Agent Runtime (Claude, Cursor, Codex, custom)
    │
    │  MCP protocol (stdio) or REST API (HTTPS)
    ▼
Naive Primitives API (api.usenaive.ai)
    │
    │  Authenticated, rate-limited, scoped to your company + agent
    ▼
Real-world services
    │
    ▼
Resend, bundle.social, HeyGen, Submagic, Pexels,
Tavily, Composio, Stripe Issuing, Twilio, etc.
```

Your agent authenticates with a single API key. The key resolves to a specific agent and company — all resources (email addresses, social accounts, payment cards, credentials) are scoped automatically. You don't manage infrastructure; you just call primitives.

## Three Ways to Connect

<Tabs>
  <Tab title="MCP Server">
    The **Model Context Protocol** server exposes Naive's primitives as tools that any MCP-compatible runtime can call directly. This is the recommended approach for Claude Desktop, Cursor, and other MCP runtimes.

    ```bash
    npm install -g @usenaive/sdk
    naive init
    ```

    The MCP server registers **12 tools** in the agent's context: 8 core primitives with full schemas, plus 4 meta-tools (`naive_context`, `naive_search`, `naive_tool_help`, `naive_execute`) for identity, discovery, and calling the remaining tools without bloating context.
  </Tab>
  <Tab title="REST API">
    Call any primitive directly over HTTPS with Bearer token authentication. Works with any language, framework, or custom agent runtime.

    ```bash
    curl -X POST https://api.usenaive.ai/v1/email/send \
      -H "Authorization: Bearer nv_sk_your_key" \
      -H "Content-Type: application/json" \
      -d '{"to":"user@example.com","subject":"Hello","body":"Welcome!","reason":"onboarding"}'
    ```
  </Tab>
  <Tab title="CLI">
    The `naive` CLI handles setup, configuration, and tool exploration from your terminal.

    ```bash
    naive status          # Connection status and agent info
    naive tools           # List all 54 tools by category
    naive tools help send_email   # Full docs for any tool
    ```
  </Tab>
</Tabs>

## The Platform (Open Source)

Naive's platform — the managed experience at [app.usenaive.ai](https://app.usenaive.ai) — lets users select and run hosted agent runtimes through a GPT-like chat interface. The runtime connects to Naive's primitives MCP, giving agents the ability to create and run companies, hire AI employees, and execute business tasks autonomously.

The platform is being open-sourced. It combines:

- **Paperclip** — the control plane (companies, agents, tasks, deliverables, billing)
- **Hermes** — the agent runtime (Claude Agent SDK, session management, tool orchestration)
- **Naive MCP** — the primitives layer (this documentation)

You can use the primitives API on its own (connect your own agent to our MCP) or use the full platform for the managed experience. Either way, Naive is only responsible for the primitives — the infrastructure that lets agents interact with the real world.
