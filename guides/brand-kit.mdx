Brand Kit is the identity primitive for your agent's company. It centralizes tagline, color palette, style direction, font, tone, and logo into a single persisted object that downstream tools and deliverables reference for visual and voice consistency.

## Tools

| Tool | Type | Description |
|------|------|-------------|
| `recommend_brand_kit` | Core | AI-generate a brand identity from company context |
| `read_brand_kit` | Extended | Get the current persisted brand kit |
| `update_brand_kit` | Extended | Update brand kit fields (partial updates supported) |
| `generate_brand_logo` | Extended | AI-generate logo SVG options |

## Why Brand Kit Matters

Without a centralized brand kit, agents make inconsistent design and voice decisions across deliverables — landing pages use different colors than social posts, email copy doesn't match the website tone. Brand Kit solves this:

- **Visual consistency** — Colors, style, and font flow into every generated deliverable
- **Voice alignment** — Tone and tagline keep copy on-brand across channels
- **Logo management** — Generate, select, and persist logos alongside the palette
- **AI-powered bootstrap** — Go from zero to complete brand identity in one tool call

## Brand Kit Lifecycle

```
Recommend → Review → Persist → Generate Logos → Select → Save Logo
                       ↑                                    |
                       └──── Update individual fields ──────┘
```

<Steps>
  <Step title="Generate a brand recommendation">
    Use `recommend_brand_kit` to AI-generate a complete brand identity. All inputs are optional — the endpoint infers from the company profile if no overrides are provided.

    <CodeGroup>
    ```bash curl
    curl -X POST https://api.usenaive.ai/v1/brand-kit/recommend \
      -H "Authorization: Bearer nv_sk_your_key_here" \
      -H "Content-Type: application/json" \
      -d '{
        "industry": "fintech",
        "target_audience": "developers",
        "style_preferences": "minimal, dark theme"
      }'
    ```

    ```python Python
    import requests

    response = requests.post(
        "https://api.usenaive.ai/v1/brand-kit/recommend",
        headers={"Authorization": "Bearer nv_sk_your_key_here"},
        json={
            "industry": "fintech",
            "target_audience": "developers",
            "style_preferences": "minimal, dark theme",
        },
    )
    brand = response.json()
    print(brand["brandKit"]["tagline"])
    print(brand["brandKit"]["colors"])
    ```

    ```typescript TypeScript

        style_preferences: "minimal, dark theme",
      }),
    });
    const { brandKit } = await response.json();
    ```

## Error Handling

| Error | Cause | Recovery |
|-------|-------|----------|
| `no_profile` | No company profile found for AI generation | Create a company profile first, or provide `company_description` and `industry` inputs |
| `generation_failed` | AI brand generation failed | Retry, or provide more specific inputs (`industry`, `style_preferences`) |
| `invalid_colors` | Color values are not valid hex strings | Use hex format like `"#1a1a2e"` for primary, secondary, and accent |
| `no_fields` | `update_brand_kit` called with no fields | Include at least one field to update |
| `no_brand_context` | Logo generation has no brand context to work with | Run `recommend_brand_kit` or `update_brand_kit` first to establish identity |

## Typical Workflow (Agent Perspective)

```
1. recommend_brand_kit({ industry: "fintech", style_preferences: "bold, dark" })
   -> brandKit: { tagline, colors, styleDirection, fontPreference, tone }
   -> Present to user: "Here's a recommended brand identity for your company"

2. [User reviews and approves, possibly tweaking tagline or colors]

3. update_brand_kit({ tagline: "...", colors: {...}, styleDirection: "...", ... })
   -> Brand kit persisted

4. generate_brand_logo({ style: "geometric, single color" })
   -> logos: ["<svg>...</svg>", "<svg>...</svg>", "<svg>...</svg>"]
   -> Present options to user: "Pick your preferred logo"

5. update_brand_kit({ logoSvg: "<svg>...</svg>" })
   -> Selected logo saved to brand kit

6. read_brand_kit()
   -> Full brand kit available for downstream deliverables
```
