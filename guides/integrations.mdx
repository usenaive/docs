Integrations is the primitive that lets an employee reach into the rest of your stack. Naive bundles two integration systems behind one consistent API: **SaaS connectors** for SaaS integrations (Slack, GitHub, HubSpot, Linear, Notion, and 100+ others) and **CMS connectors** for content platforms (Webflow, Ghost, Shopify, Framer). Connection auth is stored at the company level so multiple employees can share the same Slack workspace or HubSpot account without juggling tokens.

## Tools

| Tool | Type | Description |
|------|------|-------------|
| `check_integrations` | Core | List all available integrations (CMS, SaaS, browser) |
| `get_integration_tools` | Extended | Get available actions for a specific integration |
| `execute_integration` | Extended | Execute a tool on a connected integration |
| `get_integration_auth_url` | Extended | Get OAuth URL for connecting third-party services |

## Discovery

Start by checking what's available:

```bash
curl https://api.usenaive.ai/v1/integrations \
  -H "Authorization: Bearer nv_sk_your_key"
```

**Response:**

```json
{
  "integrations": {
    "cms": [
      { "id": "int_abc", "type": "webflow", "name": "Acme Marketing Site", "status": "connected" },
      { "id": "int_def", "type": "ghost", "name": "Acme Blog", "status": "connected" }
    ],
    "saas": [
      { "toolkit": "SLACK", "status": "connected" },
      { "toolkit": "GITHUB", "status": "connected" },
      { "toolkit": "HUBSPOT", "status": "available" }
    ],
    "connections": [
      { "id": "conn_1", "platform_name": "LinkedIn", "status": "connected" }
    ]
  }
}
```

## Workflow

All integrations follow the same three-step pattern: discover, inspect, execute.

<Steps>
  <Step title="Check available integrations">
    ```bash
    check_integrations()
    ```
    Returns all CMS, SaaS, and browser integrations with their current status.
  </Step>

  <Step title="Get available tools for an integration">
    ```bash
    # For a CMS integration
    curl "https://api.usenaive.ai/v1/integrations/tools?integration_type=cms&integration_id=int_abc" \
      -H "Authorization: Bearer nv_sk_your_key"

    # For a SaaS toolkit
    curl "https://api.usenaive.ai/v1/integrations/tools?integration_type=saas&toolkit=SLACK" \
      -H "Authorization: Bearer nv_sk_your_key"
    ```

    Response:
    ```json
    {
      "tools": [
        { "name": "SLACK_SEND_MESSAGE", "description": "Send a message to a Slack channel", "parameters": { ... } },
        { "name": "SLACK_LIST_CHANNELS", "description": "List Slack channels", "parameters": { ... } }
      ]
    }
    ```
  </Step>

  <Step title="Execute a tool">
    ```bash
    curl -X POST https://api.usenaive.ai/v1/integrations/execute \
      -H "Authorization: Bearer nv_sk_your_key" \
      -H "Content-Type: application/json" \
      -d '{
        "integration_type": "saas",
        "toolkit": "SLACK",
        "tool_name": "SLACK_SEND_MESSAGE",
        "params": {
          "channel": "#general",
          "text": "Weekly metrics report is ready!"
        }
      }'
    ```
  </Step>
</Steps>

### Connecting SaaS Integrations

To connect a new SaaS integration, get the OAuth authorization URL:

```bash
curl https://api.usenaive.ai/v1/integrations/auth-url \
  -H "Authorization: Bearer nv_sk_your_key"
```

Response:
```json
{
  "url": "https://usenaive.ai/connect/auth/abc123"
}
```

Share this URL with the user. They'll authorize the integration through the OAuth flow, and it will appear in `check_integrations` once connected.

## CMS Integrations

Native connectors for content management systems. CMS integrations support both reading and writing content.

### Supported Platforms

| Platform | Read | Write | Notes |
|----------|------|-------|-------|
| Webflow | Yes | **Draft only** | Collections, pages, CMS items |
| Ghost | Yes | **Draft only** | Posts, pages, tags |
| Shopify | Yes | **Draft only** | Products, collections, blog posts |
| Framer | Yes | **Draft only** | Pages, CMS collections |

<Info>
**All CMS writes create drafts.** Nothing is published directly. This gives the user a chance to review content before it goes live.
</Info>

### CMS Example: Create a Blog Post

```bash
curl -X POST https://api.usenaive.ai/v1/integrations/execute \
  -H "Authorization: Bearer nv_sk_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "integration_type": "cms",
    "integration_id": "int_def",
    "tool_name": "create_post",
    "params": {
      "title": "10 Ways AI is Transforming Marketing",
      "content": "<p>Artificial intelligence is reshaping how teams approach...</p>",
      "tags": ["AI", "Marketing", "Automation"]
    }
  }'
```

The post is created as a **draft** in Ghost — the user publishes it when ready.

## Tools Reference

| Tool | Method | Path | Description |
|------|--------|------|-------------|
| `check_integrations` | GET | `/v1/integrations` | List all available integrations (core tool) |
| `get_integration_tools` | GET | `/v1/integrations/tools` | List tools for a specific integration |
| `execute_integration` | POST | `/v1/integrations/execute` | Execute a tool on an integration |
| `get_integration_auth_url` | GET | `/v1/integrations/auth-url` | Get OAuth URL for connecting new integrations |

### execute_integration Parameters

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `integration_type` | string | Yes | `cms` or `saas` |
| `tool_name` | string | Yes | Tool to execute |
| `params` | object | No | Tool parameters (default `{}`) |
| `integration_id` | string | Conditional | Required for CMS integrations |
| `toolkit` | string | Conditional | Required for SaaS integrations |

## Error Handling

| Error | Cause | Recovery |
|-------|-------|----------|
| `integration_not_connected` | Target integration not yet connected | Use `get_integration_auth_url` to get the OAuth URL |
| `tool_not_found` | Specified tool doesn't exist for this integration | Call `get_integration_tools` to list available tools |
| `auth_expired` | Integration OAuth token expired | Re-authorize via the integrations portal |

## Typical Agent Flow

```
Task: "Post our new blog article to Ghost and share it on Slack"

1. check_integrations()
   → Ghost CMS connected, Slack connected

2. get_integration_tools({ integration_type: "cms", integration_id: "int_def" })
   → Available: create_post, list_posts, get_post, ...

3. execute_integration({
     integration_type: "cms",
     integration_id: "int_def",
     tool_name: "create_post",
     params: { title: "...", content: "..." }
   })
   → Draft created in Ghost

4. execute_integration({
     integration_type: "saas",
     toolkit: "SLACK",
     tool_name: "SLACK_SEND_MESSAGE",
     params: { channel: "#content", text: "New blog draft ready for review: ..." }
   })
   → Slack notification sent
```
