---
title: "Get Verification Status"
description: "Check the current KYC verification status for a member."
api: "GET /v1/verification/:memberId/status"
---

<ParamField path="memberId" type="string" required>
  The member ID to look up verification status for.
</ParamField>

<CodeGroup>
```bash curl
curl https://api.usenaive.ai/v1/verification/mem_xyz789/status \
  -H "Authorization: Bearer nv_sk_your_key_here"
```

```python Python
import requests

response = requests.get(
    "https://api.usenaive.ai/v1/verification/mem_xyz789/status",
    headers={"Authorization": "Bearer nv_sk_your_key_here"},
)
data = response.json()
```

```typescript TypeScript
const response = await fetch("https://api.usenaive.ai/v1/verification/mem_xyz789/status", {
  headers: { "Authorization": "Bearer nv_sk_your_key_here" },
});
const data = await response.json();
```
</CodeGroup>

<ResponseField name="member_id" type="string">
  The member this status belongs to.
</ResponseField>

<ResponseField name="verification_id" type="string | null">
  Internal vault user ID. `null` until the member completes the hosted flow.
</ResponseField>

<ResponseField name="kyc_status" type="string">
  Verification status. One of `pending`, `passed`, `failed`, `review`, `revoked`.
</ResponseField>

<ResponseField name="watchlist_hit_count" type="number">
  Number of watchlist matches found during screening.
</ResponseField>

<ResponseField name="document_uploaded" type="boolean">
  Whether the member uploaded an identity document.
</ResponseField>

<ResponseField name="last_updated_at" type="string">
  ISO 8601 timestamp of the last status change.
</ResponseField>
