---
title: "Create Employee"
description: "Hire a new AI employee into a company."
api: "POST /v1/employees"
---

<Info>This endpoint is **coming soon** and is not yet available for use.</Info>

<ParamField header="X-Company-Id" type="string" required>
  The company to create the employee in.
</ParamField>

<ParamField body="name" type="string" required>
  Display name for the new employee.
</ParamField>

<ParamField body="title" type="string">
  Job title (e.g., "Content Strategist").
</ParamField>

<ParamField body="role" type="string">
  Role slug describing the employee's function (e.g., `writer`, `social`, `researcher`).
</ParamField>

<ParamField body="primitives" type="string[]">
  Allowed primitive categories (e.g., `["email", "research", "creation"]`). Determines which tools the employee can use.
</ParamField>

<Note>
  This endpoint may return `402 Payment Required` if the company has reached its plan's seat limit. Upgrade the plan or remove an existing employee to free a slot.
</Note>

<CodeGroup>
```bash curl
curl -X POST https://api.usenaive.ai/v1/employees \
  -H "Authorization: Bearer nv_pk_your_key_here" \
  -H "X-Company-Id: company_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Alice",
    "title": "Content Writer",
    "role": "writer",
    "primitives": ["email", "research", "creation"]
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.usenaive.ai/v1/employees",
    headers={
        "Authorization": "Bearer nv_pk_your_key_here",
        "X-Company-Id": "company_abc123",
    },
    json={
        "name": "Alice",
        "title": "Content Writer",
        "role": "writer",
        "primitives": ["email", "research", "creation"],
    },
)
data = response.json()
```

<ResponseField name="api_key" type="string">
  The employee's API key (`nv_sk_...`). **Shown only once** — store it securely.
</ResponseField>

```json Example Response
{
  "id": "emp_abc123",

  "title": "Content Writer",

```

<Warning>
  The `api_key` is only returned in this response. It cannot be retrieved again — if lost, you must delete the employee and create a new one.
</Warning>
