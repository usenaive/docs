---
title: "Resend API Proxy"
description: "Proxy any Resend API call through Naive. Access contacts, segments, broadcasts, templates, batch emails, and more."
api: "POST /v1/resend"
---

Forward any [Resend API](https://resend.com/docs/api-reference) request through Naive's managed infrastructure. Your agent gets full access to Resend's API without managing API keys directly.

<ParamField body="method" type="string" required>
  HTTP method: `GET`, `POST`, `PATCH`, or `DELETE`.
</ParamField>

<ParamField body="path" type="string" required>
  Resend API path (e.g. `/contacts`, `/emails/batch`, `/segments/seg_abc123`). Must start with `/`.
</ParamField>

<ParamField body="body" type="object">
  Request body for POST/PATCH requests. Passed as JSON to the Resend API.
</ParamField>

<Warning>
  Domain management paths (`/domains`) are blocked — use Naive's dedicated [domain primitives](/api-reference/domains/register-domain) instead.
</Warning>

## Available Endpoints

| Category | Endpoints | Resend Docs |
|----------|-----------|-------------|
| Emails | `POST /emails`, `POST /emails/batch`, `GET /emails/:id` | [Emails](https://resend.com/docs/api-reference/emails) |
| Contacts | `POST/GET/PATCH/DELETE /contacts` | [Contacts](https://resend.com/docs/api-reference/contacts) |
| Contact Properties | `POST/GET/PATCH/DELETE /contact-properties` | [Properties](https://resend.com/docs/api-reference/contacts/contact-properties) |
| Segments | `POST/GET/DELETE /segments`, `GET /segments/:id/contacts` | [Segments](https://resend.com/docs/api-reference/segments) |
| Topics | `POST/GET/PATCH/DELETE /topics` | [Topics](https://resend.com/docs/api-reference/topics) |
| Broadcasts | `POST/GET/PATCH/DELETE /broadcasts` | [Broadcasts](https://resend.com/docs/api-reference/broadcasts) |
| Templates | `POST/GET/PATCH/DELETE /templates` | [Templates](https://resend.com/docs/api-reference/templates) |

## Examples

### Create a contact

<CodeGroup>
```bash curl
curl -X POST https://api.usenaive.ai/v1/resend \
  -H "Authorization: Bearer nv_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "method": "POST",
    "path": "/contacts",
    "body": {
      "email": "steve@example.com",
      "first_name": "Steve",
      "last_name": "Wozniak"
    }
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.usenaive.ai/v1/resend",
    headers={"Authorization": "Bearer nv_sk_your_key_here"},
    json={
        "method": "POST",
        "path": "/contacts",
        "body": {
            "email": "steve@example.com",
            "first_name": "Steve",
            "last_name": "Wozniak",
        },
    },
)
data = response.json()
```

```typescript TypeScript
const response = await fetch("https://api.usenaive.ai/v1/resend", {
  method: "POST",
  headers: {
    "Authorization": "Bearer nv_sk_your_key_here",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    method: "POST",
    path: "/contacts",
    body: {
      email: "steve@example.com",
      first_name: "Steve",
      last_name: "Wozniak",
    },
  }),
});
const data = await response.json();
```

```javascript JavaScript

  method: "POST",
  headers: {
    "Authorization": "Bearer nv_sk_your_key_here",

      email: "steve@example.com",

      last_name: "Wozniak",
    },
  }),
});
const data = await response.json();
```
</CodeGroup>

### List all contacts

<CodeGroup>
```bash curl
curl -X POST https://api.usenaive.ai/v1/resend \
  -H "Authorization: Bearer nv_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"method": "GET", "path": "/contacts"}'
```

```python Python
response = requests.post(
    "https://api.usenaive.ai/v1/resend",
    headers={"Authorization": "Bearer nv_sk_your_key_here"},
    json={"method": "GET", "path": "/contacts"},
)
contacts = response.json()["data"]
```

```typescript TypeScript
const response = await fetch("https://api.usenaive.ai/v1/resend", {
  method: "POST",
  headers: {
    "Authorization": "Bearer nv_sk_your_key_here",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ method: "GET", path: "/contacts" }),
});
const { data: contacts } = await response.json();
```

```javascript JavaScript
const response = await fetch("https://api.usenaive.ai/v1/resend", {
  method: "POST",
  headers: {
    "Authorization": "Bearer nv_sk_your_key_here",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ method: "GET", path: "/contacts" }),
});
const { data: contacts } = await response.json();
```
</CodeGroup>

<Info>
For simple single-email sends, prefer the dedicated [send_email](/api-reference/communication/send-email) endpoint — it handles agent identity, sensitive data blocking, and audit logging automatically. Use the Resend proxy for everything else: contacts, segments, broadcasts, templates, batch sends, and delivery tracking.
</Info>
