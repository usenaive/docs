## Base URL

```
https://api.usenaive.ai
```

All endpoints are prefixed with `/v1/`.

## Authentication

Most endpoints require a Bearer token. Include your API key in the `Authorization` header:

```bash
curl https://api.usenaive.ai/v1/status \
  -H "Authorization: Bearer nv_sk_your_key_here"
```

API keys are scoped to a specific agent and company. The key automatically resolves identity — you never need to pass `agent_id`.

## Rate Limits

Requests are rate-limited per API key. If you exceed the limit, you'll receive a `429 Too Many Requests` response with a `Retry-After` header.

| Tier | Requests/min |
|------|-------------|
| Free | 60 |
| Pro | 300 |

## Error Format

All errors follow a consistent JSON structure:

```json
{
  "error": {
    "code": "rate_limited",
    "message": "Too many requests. Retry after 30 seconds."
  }
}
```

Common error codes:

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `unauthorized` | 401 | Missing or invalid API key |
| `forbidden` | 403 | Key doesn't have access to this resource |
| `not_found` | 404 | Resource doesn't exist |
| `rate_limited` | 429 | Too many requests |
| `internal_error` | 500 | Server error |
